A virada de chave

Claude no chat  vs  Claude Code

É o mesmo cérebro. O que muda é onde ele trabalha e o que ele consegue tocar.

💬 Claude (chat)

  • Você conversa, ele responde.
  • Ele te diz o código, você copia e cola na mão.
  • Não enxerga teus arquivos nem teu projeto.
  • Ótimo pra pensar, escrever, tirar dúvida.
  • Cada coisa é um copia-e-cola seu.
→

⌨️ Claude Code

  • Vive dentro do teu computador, na pasta do projeto.
  • Ele cria e edita os arquivos sozinho.
  • Roda comandos, instala coisas, sobe o app.
  • Lê o projeto inteiro pra entender o contexto.
  • Você pede em português, ele faz.

Resumo: o chat é um consultor brilhante do outro lado da mesa. O Claude Code é um dev que senta na sua máquina e põe a mão na massa.

🧠

🗣️

↩️

Setup · passo a passo

Preparando o terreno

  • 1
    Conta no Claude — crie em claude.ai. O Claude Code usa a assinatura (plano Pro/Max) ou uma chave de API.
  • 2
    Instale o Node.js — é o motor que roda as ferramentas. Baixe a versão LTS em nodejs.org e instale (next, next, next).
  • 3
    Instale o Claude Code — um comando só, no terminal:
terminal
npm install -g @anthropic-ai/claude-code
  • 4
    Entre numa pasta e ligue — crie uma pasta pro projeto, abra o terminal nela e digite claude. Na primeira vez ele pede pra você logar na conta. Pronto.

Dica: confirme sempre a versão mais recente e o método oficial em claude.com/claude-code — a instalação evolui rápido.

As ferramentas do nosso processo

Os 4 nomes que você vai ouvir

Pra sair de uma ideia até um app no ar, a gente usa quatro peças. Cada uma com um papel simples:

🐙

GitHub

O cofre do projeto. Guarda todo o código e cada versão dele. É o seu backup e seu histórico — dá pra voltar a qualquer ponto.

🗄️

Supabase

O banco de dados e o login. É onde o app guarda as informações (usuários, cadastros) e controla quem entra.

▲

Vercel

A publicação. Pega o seu app e coloca no ar com um link, pra qualquer pessoa abrir no navegador.

📱

PWA

O truque do app. Faz o site virar um aplicativo instalável no celular, com ícone na tela, abrindo em tela cheia.

Boa notícia: o Claude Code conversa com todas elas. Você não precisa virar especialista em nenhuma — só entender o que cada uma faz.

Dando as chaves pro Claude Code

Conectar tudo pelo terminal

Você loga uma vez só em cada serviço, ali no terminal. A partir daí o Claude Code usa GitHub, Vercel e Supabase sozinho — sobe código, publica e configura o banco sem você clicar em nada.

🐙

GitHub

Conta: github.com → Sign up

terminal
gh auth login
▲

Vercel

Conta: vercel.com → Sign up (entra com o GitHub)

terminal
npm i -g vercel && vercel login
🗄️

Supabase

Conta: supabase.com → Start project (entra com o GitHub)

terminal
npm i -g supabase && supabase login

Cada comando abre o navegador pra você confirmar o login e pronto. O do GitHub precisa do gh instalado antes (Mac: brew install gh · Windows: winget install GitHub.cli). Travou em algo? Cola no Claude Code: "me guie passo a passo pra instalar e logar o gh, a Vercel e o Supabase no terminal, explicando como pra um iniciante."

🖼️ Primeiro o visual (protótipo)

A gente monta todas as telas com dados de mentira primeiro. Você navega, sente o app, ajusta — sem mexer em banco nenhum. Rápido e barato de mudar.

🔌 Depois liga de verdade

Com o visual aprovado, aí sim a gente troca os dados de mentira pelos reais (Supabase): login, cadastros, tudo funcionando.

Por que essa ordem? Porque mudar um botão de lugar no protótipo custa 1 minuto. Mudar depois de tudo conectado custa uma tarde.

A arma secreta do visual

Bonito sem ser designer

Tem uma skill (uma habilidade extra que se instala no Claude Code) chamada ui-ux-pro-max. Ela carrega estilos, paletas de cor, combinações de fonte e regras de boa interface. Quando você pede uma tela, o Claude consulta isso e entrega algo com cara de produto de verdade, não de trabalho de escola.

🎨

50+ estilos

De minimalista a glass, dark mode, bento grid…

🌈

161 paletas

Combinações de cor que funcionam juntas.

🔤

57 pares de fonte

Tipografia que combina, escolhida pra você.

Na hora de criar a tela, basta pedir: "use a skill ui-ux-pro-max, estilo minimalista dark estilo Apple" — e o nível do visual sobe na hora.

O pulo do gato

Transformar o site em app de celular

PWA é o que faz o seu projeto deixar de ser "um site" e virar um app instalável: ícone na tela inicial, abre em tela cheia, sem a barra do navegador. Parece nativo.

📲

Instala sem loja

O usuário abre o link e toca em "Adicionar à tela de início". Sem App Store, sem Play Store.

🍎

Cara de app nativo

Ícone, splash de abertura, tela cheia, gestos. A gente segue as diretrizes da Apple pra parecer um app de iPhone de verdade.

É só pedir pro Claude Code: "deixa esse app instalável como PWA, com ícone e splash, pronto pra adicionar na tela do celular".

Caixa de ferramentas · prompts prontos

1 · Criar o projeto

cole no Claude Code
Crie um novo projeto de aplicativo web chamado "<NOME DO APP>".

Stack (siga exatamente):
- React 19 + Vite + Tailwind CSS v4 + JavaScript (sem TypeScript)
- Estrutura de pastas: src/pages (uma tela por arquivo), src/components/ui
  (botões, inputs reutilizáveis), src/lib (integrações), src/hooks
- Mobile-first, pronto pra virar PWA depois
- No index.html, o <meta charset="utf-8"> tem que ser a PRIMEIRA linha do <head>

Comece criando o projeto, instalando as dependências e fazendo uma tela inicial
simples com o nome do app. Depois ligue o app pra eu ver no navegador.

Esse prompt já embute a "receita da casa": ele nasce no padrão certo, sem você precisar saber os detalhes.

2 · Gerar as telas (com design)

cole no Claude Code
Use a skill ui-ux-pro-max, estilo minimalista dark inspirado na Apple.

Crie as telas do MVP deste app: <LISTE AS TELAS, ex: Login, Lista de gastos,
Adicionar gasto, Resumo do mês>.

Regras:

🎁 Bônus exclusivo da turma

O agente que tira a ideia da sua cabeça

Nem sempre a ideia já vem redonda. Esse agente faz uma entrevista leve com você (a partir de um áudio ou texto) e devolve um escopo pronto — incluindo um prompt pra colar no Claude Code e já começar a construir. É o mesmo método de discovery que a gente usa nos clientes, destilado pra você.

cole numa conversa nova do Claude
Você é um parceiro de discovery de produto. Seu trabalho é pegar uma ideia crua
de aplicativo que a pessoa vai descrever (texto ou transcrição de um áudio) e,
através de uma entrevista leve, transformar em um escopo claro e construível.
A pessoa NÃO é técnica. Fale simples, sem jargão, com paciência e bom humor.

Conduza assim:
1. Apresente-se em 1 linha e peça pra pessoa contar a ideia com as palavras dela.
2. Faça UMA pergunta de cada vez. Nunca despeje um questionário. Reaja ao que ela
   disse antes de seguir.
3. Ao longo da conversa, descubra: a dor real (o que ela faz hoje sem o app);
   quem usa e em que momento; o "loop de valor" mínimo (a menor sequência de ações
   que já entrega valor); o momento "uau"; o que o app NÃO é; e como ela saberia em
   30 dias que valeu a pena.
4. Quando achar uma contradição ou uma premissa arriscada, aponte com gentileza.
5. Em 5 a 8 trocas você já tem o suficiente. Aí diga "posso fechar o escopo?".

No fim, entregue um documento limpo com: o app em uma frase; o problema; quem usa;
o loop de valor mínimo; o que entra no MVP (marcando MUST/SHOULD/COULD); o que fica
de fora de propósito; riscos e premissas; e — o mais importante — um "Prompt pronto
pro Claude Code" em português, concreto o bastante pra colar e ver telas aparecendo,
começando pela prototipação visual com dados de exemplo, mobile-first.

Comece agora se apresentando e fazendo a primeira pergunta.
